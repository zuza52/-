# Captcha Everything — exteraGram plugin

Перед каждой отправкой сообщения плагин показывает капчу «выбери правильную
картинку». Пока ты не ткнёшь нужный эмодзи — сообщение не уйдёт.

## Файлы

- `captcha_everything.plugin.py` — сам плагин для exteraGram (Python SDK).

## Установка

1. Открой exteraGram на Android.
2. Настройки → exteraGram → Плагины → «Установить из файла».
3. Выбери `captcha_everything.plugin.py`.
4. Включи плагин в списке.

Плагин требует:
- exteraGram `>= 12.5.1`
- Plugin SDK `>= 1.4.3.6`

## Как это работает

- Хук `on_send_message_hook` перехватывает каждое исходящее сообщение.
- Отправка отменяется (`HookStrategy.CANCEL`), на UI-потоке показывается
  диалог с вопросом «Pick: cat» и 3–5 эмодзи-кнопками.
- При правильном ответе сообщение повторно отправляется через
  `client_utils.send_message`. При этом мы помечаем его как
  «предварительно одобренное», чтобы хук пропустил его без повторной капчи.
- При неправильном ответе — ничего не уходит, показывается тост.

## Настройки (внутри плагина)

- **Enable captcha on send** — вкл/выкл.
- **Number of options** — 3, 4 или 5 эмодзи на выбор.
- **Show toast on wrong answer** — показывать подсказку с правильным ответом
  после ошибки.

## Известные ограничения

- **Нет капчи на «каждую букву».** Плагин SDK не даёт доступа к полю ввода
  чата (`TextWatcher`) без лазания в приватные классы клиента, поэтому
  минимальная гранулярность — «одно отправленное сообщение = одна капча».
  Если очень нужно раздражать себя на каждый символ — это делается только
  через реверс внутренностей ChatActivity и будет ломаться с каждым
  обновлением exteraGram.
- Капча использует эмодзи, а не изображения-ресурсы: ресурсные картинки
  в `AlertDialog` из плагина SDK не прокинуть. Эмодзи выглядят одинаково
  на всех современных Android и визуально работают как «картинки».
- После правильного ответа сообщение отправляется повторно, поэтому
  отметка «Typing…» у собеседника может мигнуть дважды.
