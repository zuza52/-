# Captcha Everything — exteraGram plugin

Перед каждой отправкой сообщения плагин показывает капчу «выбери правильную
картинку». Пока ты не ткнёшь нужный эмодзи — сообщение не уйдёт.

## Файлы

- `captcha_everything.plugin.py` — сам плагин для exteraGram (Python SDK).

## Установка

1. Открой exteraGram на Android.
2. Настройки → exteraGram → Плагины → «Установить из файла».
3. Выбери `captcha_everything.plugin.py`.
4. Включи плагин в списке.

## Как это работает (v1.2.0)

- Хук `on_send_message_hook` перехватывает исходящее сообщение.
- Поток отправки **блокируется** до ответа на капчу.
- На UI-потоке показывается диалог «Pick: cat» с 3–5 эмодзи-кнопками.
- Правильный эмодзи → хук возвращает `HookStrategy.DEFAULT`, сообщение
  уходит **сразу же**, без повторного нажатия «Отправить».
- Неправильный или отмена → `HookStrategy.CANCEL`, сообщение не уходит.
- Таймаут 60 секунд без ответа → тоже отмена.

История версий:
- **v1.0.0** — пыталась переотправлять сообщение через `client_utils.send_message`,
  но такой функции в SDK нет → сообщения пропадали.
- **v1.1.0** — одноразовый «pass» на 30 секунд; правильный ответ разблокировал
  следующую отправку, надо было жать «Отправить» дважды.
- **v1.2.0** — блокирующая капча: поток отправки ждёт ответа и отпускает его
  сам. Одно нажатие «Отправить» = одно сообщение.

## Настройки (внутри плагина)

- **Enable captcha on send** — вкл/выкл.
- **Number of options** — 3, 4 или 5 эмодзи на выбор.
- **Show toast on wrong answer** — показывать подсказку с правильным ответом
  после ошибки.

## Известные ограничения

- **Нет капчи на «каждую букву».** Плагин SDK не даёт доступа к полю ввода
  чата (`TextWatcher`) без лазания в приватные классы клиента, поэтому
  минимальная гранулярность — «одно отправленное сообщение = одна капча».
  Если очень нужно раздражать себя на каждый символ — это делается только
  через реверс внутренностей ChatActivity и будет ломаться с каждым
  обновлением exteraGram.
- Капча использует эмодзи, а не изображения-ресурсы: ресурсные картинки
  в `AlertDialog` из плагина SDK не прокинуть. Эмодзи выглядят одинаково
  на всех современных Android и визуально работают как «картинки».
- Капча блокирует поток, в котором exteraGram вызывает хук. Если этот поток
  окажется главным (UI) — интерфейс подвиснет, пока не ответишь. На всех
  актуальных версиях хук вызывается с фонового потока, так что в норме
  этого не происходит, но если столкнёшься — крикни, переделаю на
  неблокирующий вариант с ручным ре-сендом.
